# Improvements & Roadmap

kontexta Publish is designed to be extensible and continuously improving. Here's what's planned and how you can contribute.

## Current Capabilities

kontexta Publish already provides:

- ✅ Single-file static site generation
- ✅ Three-pane responsive layout
- ✅ Client-side search with type filtering
- ✅ Mermaid diagram rendering with zoom/fullscreen
- ✅ API endpoint documentation with interactive modals
- ✅ Glossary term management
- ✅ Dark/light theme toggle
- ✅ Hash-based routing and scroll-spy
- ✅ kontexta theme tokens and typography

## Planned Improvements

### Phase 1: Enhanced Navigation

- **Breadcrumb navigation** — Show current location in the document hierarchy
- **Previous/Next links** — Navigate between documents sequentially
- **Document versioning** — Support multiple versions of documentation
- **Multi-language support** — i18n for international documentation

### Phase 2: Search Enhancements

- **Fuzzy search** — Tolerate typos and partial matches
- **Search filters** — Filter by folder, date, or tags
- **Search analytics** — Track popular searches and missing content
- **External search integration** — Connect to Algolia or similar services

### Phase 3: Rendering Improvements

- **Custom themes** — Allow users to define their own color schemes
- **Plugin system** — Extend rendering with custom block types
- **Math rendering** — Support LaTeX equations via KaTeX
- **PDF export** — Generate printable PDF versions

### Phase 4: Developer Experience

- **Hot reload** — Instant preview during development
- **CLI improvements** — More configuration options
- **CI/CD integration** — Automated documentation builds
- **Documentation** — Comprehensive user guide and examples

## Contributing

We welcome contributions! Here's how you can help:

### Reporting Issues

- Use the GitHub issue tracker
- Include steps to reproduce
- Mention your browser and OS version
- Provide screenshots if applicable

### Submitting Pull Requests

1. Fork the repository
2. Create a feature branch
3. Write tests for new functionality
4. Ensure all tests pass
5. Submit a pull request with a clear description

### Code Style

- TypeScript with strict mode
- ESLint with recommended rules
- Prettier for formatting
- Vitest for testing

## Feedback

We'd love to hear from you! Share your thoughts on:

- What features would you like to see?
- Are there any bugs you've encountered?
- How are you using kontexta Publish?

Your feedback helps us prioritize improvements and make kontexta Publish better for everyone.
